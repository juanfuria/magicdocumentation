Welcome Windows Mobile Developers

The Handpoint Handpoint platform a.k.a headstart on Windows Mobile/CE provides an application programming interface simplifying the development of your payment solutions considerably.

You as a POS application developer can link your C++ based app to our heft.dll for integration with Handpoint on a Windows Mobile/CE platform.

The SDK includes:

  • Sample apps:  A WTL (Windows Template Library) based Windows Mobile/CE sample applications .
  • Sample app source code: Giving you an indication on how to use our library.
  • Documentation: Doxygen documentation which can be viewed in browser.
  • Libs: C++ .lib and .dll files for development.
  • Header/include files: C++ header files with function prototypes, structs and defines needed for integration.

Heft module is packaged as Dynamic linked library named heft.dll. Its name is an abbreviation of Handpoint Electronic Funds Transfer. It contains pure C++ implementation of EFT (Electronic Funds Transfer) protocol, which is used for communication between Handpoint library and PED device.

Central part of Handpoint Windows Mobile/CE solution is the Handpoint Library. This library provides core functionality, including interaction with PED device and communication with payment gateway. Those integrators who wish to integrate Handpoint solution into their C++ based products, will most likely use Handpoint Library directly from their C++ based applications for these purposes.
